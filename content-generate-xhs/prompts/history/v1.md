## 版本说明
初始版本，基于 CodeWiki.md 的设计规范创建。

## 设计目标

本 Prompt 的设计目标是为小红书图文文案生成提供一个完整的提示词框架，确保生成的内容符合以下要求：
- 语言年轻、真诚、不油腻，像朋友聊天一样自然
- 开头有钩子，能在3秒内抓住读者注意力
- 内容有干货，提供可操作的实际价值
- 段落短，适合手机阅读
- 标题吸引人，标签精准

## 核心设计思路

### 1. 系统提示词 (system.md)

系统提示词采用"角色设定 + 创作原则 + 输出格式 + 结构建议 + 禁忌清单"的五层结构：

#### 角色设定
- 定位为"资深小红书内容创作者"
- 强调擅长"年轻、真诚、有干货"的内容风格
- 明确任务是"根据选题和素材生成图文笔记"

#### 创作原则（6条）
| 原则 | 说明 | 设计理由 |
|------|------|---------|
| 语言风格 | 年轻、真诚、接地气 | 小红书用户偏好自然不做作的内容 |
| 开头钩子 | 前3秒抓住注意力 | 小红书信息流模式下，开头决定点击率 |
| 内容干货 | 可操作的方法、真实经验 | 避免空泛，提供实际价值 |
| 表达克制 | 不用绝对化词汇 | 避免营销感，增强可信度 |
| 段落节奏 | 短段落、多换行 | 适配手机阅读习惯 |
| 情感连接 | 适当加入个人感受 | 增强共鸣和互动率 |

#### 输出格式（JSON 严格约束）
- 强制要求输出合法 JSON
- 定义了6个必填字段及其类型
- 对字符串长度进行了精确约束：
  - 标题：12-25字
  - 正文：500-800字
  - 封面文案：10-12字
  - 标签：5-8个

#### 标题创作要点
- 多用数字（增加具体感和可信度）
- 用括号补充信息（增加信息量）
- 制造悬念或痛点（提升点击率）
- 包含关键词（利于搜索）

#### 正文结构建议（5段式）
1. 开头钩子（1-2句话）
2. 引入背景/痛点
3. 核心内容（分点阐述）
4. 个人经验/案例
5. 结尾总结（鼓励互动）

#### 禁忌清单（5条）
- ❌ 营销号口吻（家人们谁懂啊）
- ❌ 内容空洞
- ❌ 绝对化表达
- ❌ 段落过长
- ❌ 低俗或敏感词汇

### 2. 用户提示词模板 (user_template.md)

采用结构化输入方式，包含4个变量：

| 变量 | 说明 | 示例 |
|------|------|------|
| {topic} | 选题主题 | 开学季社团招新如何提高报名转化率 |
| {column} | 栏目分类 | 经验干货 |
| {materials} | 参考素材列表 | 用 markdown 列表格式 |
| {tone} | 语气风格 | 年轻、真诚、不油腻 |
| {audience} | 目标受众 | 大学生 |

设计理由：
- 结构化输入便于程序化处理
- 变量名语义清晰，易于理解
- 支持素材的多条输入

## 待优化点

1. **语言风格微调**：需要根据实际 LLM 生成效果调整语气描述，可能需要增加更多示例
2. **标题生成策略**：当前标题策略较通用，可考虑增加特定类型的标题模板（如数字型、悬念型、情感型）
3. **正文结构灵活性**：5段式结构可能过于僵化，可考虑根据内容类型动态调整
4. **标签策略优化**：当前仅要求数量和格式，可增加标签热度、相关性的要求
5. **封面文案指导**：可增加封面文案的设计原则（如突出痛点、制造悬念等）
