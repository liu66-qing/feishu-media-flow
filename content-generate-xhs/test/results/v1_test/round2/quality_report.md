### 2.1 test_01 — 社团招新

**选中标题**：社团招新没人来？5个实操技巧拉满氛围（亲测）

## 五、v2 Prompt 迭代方向

基于 round1 + round2 两轮测试发现的问题，v2 Prompt 迭代优先级调整如下：

1. **【高优先级】禁止 emoji/符号分点**：明确要求使用纯文字分段+过渡句（如「先说第一点」「除此之外」），禁止使用 1️⃣📌▫️• 等符号/编号，避免清单工具感
2. **【高优先级】正文字数硬约束强化**：明确要求「每个要点至少展开 2-3 句，配合个人经历或具体案例」，避免干巴巴罗列；要求模型生成后自检字数
3. **【高优先级】强制个人故事感**：在 system 中要求「必须以第一人称叙述，至少包含 1 个具体的个人经历/观察案例」，避免通用攻略
4. **【中优先级】cover_text 字数控制**：给出 10-12 字的正反示例（好：「大学恋爱走心避坑指南」10字；差：「招新」2字/「招新爆款指南实操技巧全收录」14字）
5. **【中优先级】绝对化词替换**：给出替换词表（最 → 很/相当/个人觉得很；第一 → 首先/最先要）
6. **【低优先级】标题钩子多样**：避免连续使用「5个XX」「必看」类模板化标题

## 六、结论

### 6.1 综合评价

| 评估维度 | 结论 |
|---------|------|
| 代码可运行性 | ✅ 脚本稳定，5 组全部成功返回 |
| 输出格式合规性 | ✅ 91% 字段达标，主要问题为正文长度和 cover_text 波动，不影响下游 |
| 文案内容质量 | ⚠️ 5 篇肉眼评分平均 18.2/25，≥3 分通过线，但工具号感较 round1 加重 |
| 风格符合度 | ⚠️ test_02/04 保持优秀，test_01/03/05 滑向清单体攻略风 |
| 稳定性 | ⚠️ 两轮测试风格波动明显，同一 Prompt 可产出优质聊天文（test_02）也可产出工具号清单（test_03/05），一致性需加强 |

### 6.2 本轮结论

**v1 Prompt round2 测试通过**，但质量较 round1 略有下滑，主要体现在工具号感加重、正文变短两方面。test_02（恋爱）和 test_04（社恐）依然保持高质量，说明模型在情感类话题上风格更稳定；干货类话题（招新/创业/就业）容易滑向清单体。

### 6.3 后续动作

1. 根据 §5 优化方向迭代 v2 Prompt
2. 同一 5 个选题跑 round3（即 v1 的最后一轮），验证 Prompt 调整是否有效
3. v1 三轮结束后对比分析，确定 v2 迭代方向
4. 最终结果需通过 `risk-check` Skill 做正式风险审查
