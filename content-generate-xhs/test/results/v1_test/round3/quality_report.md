## 五、v1 三轮整体结论与 v2 迭代方向

### 5.1 v1 Prompt 综合评估

v1 Prompt 经过三轮共 15 条输出测试，整体结论：

| 评估维度 | 结论 |
|---------|------|
| 格式稳定性 | ✅ 91%-95% 字段合规，JSON/退出码/字段完整性 100% 通过 |
| 内容可用性 | ✅ 15/15 篇均 ≥3 分通过线，文案可直接使用 |
| 风格一致性 | ⚠️ 存在波动：情感类话题（恋爱/社恐）稳定在 4-5 分，干货类话题（招新/创业/就业）偶尔滑向工具号清单体，三轮中最好篇目（test_02 round3，24分）与最弱篇目（test_01 round2，16分）差距明显 |
| 核心问题 | 1. cover_text 字数不稳定（9-14字波动）<br>2. 干货类话题容易使用 emoji 编号/符号分点<br>3. 正文偶尔不足 500 字 |
| 核心优势 | 1. 开头钩子普遍质量高<br>2. 情感类话题共情能力强<br>3. 建议可操作性强，有具体细节<br>4. 结尾互动引导自然 |

### 5.2 v2 Prompt 迭代方向（基于三轮数据）

**高优先级**：
1. **封面文案字数强约束+示例**：给出明确字数范围和 2-3 个 10-12 字正面示例 + 反例，要求模型生成后自检
2. **干货话题去工具化**：明确要求干货类内容也必须用「第一/第二/另外/最后」等文字过渡，禁止 emoji 编号和 •▫️ 等符号列表
3. **正文字数兜底**：要求模型生成后统计字数，若低于 500 字自动扩写（每个要点增加 1-2 句细节或个人经历）
4. **标题与内容数量一致**：要求标题中「N个XX」的 N 必须与正文实际展开段落数一致

**中优先级**：
5. **强制个人视角**：开头必须以第一人称具体经历切入（「我去年/我大一/我见过…」），禁止空泛开场
6. **绝对化词替换引导**：提供替换词表，但不做硬性禁止（因为语义合理的「最」字不影响阅读体验）

**低优先级**：
7. **参考 test_04 emoji 用法**：如果用 emoji，应作为情绪/主题图标（🌱📚🤝🏠）而非数字编号
8. **risk_notes 主动填写**：引导模型在涉及金钱、学业、感情决策等敏感话题时主动填写风险提示（参考 round3 test_03 的做法）
