## 五、v1 版本优势总结

1. **代码执行稳定**：main.py + run_round.py 自动化流程 15 次执行零失败，容错机制（LLM 响应解析重试、数组处理）有效
2. **钩子开场优秀**：5 个选题 × 3 轮共 15 篇均有高质量开场（痛点/反问/对比/场景还原），钩子强度稳定 4 分
3. **结尾互动自然**：15 篇均以「评论区聊聊」类引导收尾，互动感强，无生硬 CTA
4. **建议可操作性强**：每篇 3-5 个具体建议，有细节支撑（如「约饭请教带具体问题」「留存率是普通单页5倍」）
5. **价值观正向**：社恐篇「找到舒服的频率」、恋爱篇「高质量单身>低质量恋爱」、创业篇「稳才重要」等观点积极不焦虑
6. **段落排版合理**：均为短段落（13-36 段），适合竖屏阅读，无大段文字堆砌
7. **学习能力可见**：标题超长问题在 r1 后完全解决；正文偏短问题从 r2 的差 68-99 字收敛到 r3 的差 14 字

## 六、v2 Prompt 改进建议

基于三轮 15 条输出的测试数据，按优先级排列以下改进项：

### 6.1 高优先级改进（直接影响通过率和风格）

#### 改进 1：封面文案（cover_text）强约束 + 示例

**现状问题**：15 篇中 6 篇 cover_text 不合规（40%），是最高频硬伤，偏差多为 1-2 字。

**改进方案**：
```
【封面文案要求】
- 字数：严格 10-12 个汉字（含标点），不能多也不能少
- 位置：封面主标题，需有吸引力
- 结构建议：「主题词 + 利益点/情绪词」
- ✅ 正面示例（10-12字）：
  · 大学恋爱走心避坑指南（10字）
  · 社恐也能轻松交朋友吗（10字）
  · 社团招新转化率翻倍秘籍（11字）
  · 在校生创业避坑实战指南（11字）
  · 大一大二拉开差距真相（10字）
- ❌ 反面示例：
  · 「社团招新」（4字，太短，无信息量）
  · 「招新爆款指南｜实操技巧全收录」（14字，太长）
  · 「早规划，少走弯路」（8字，太空泛）
- 生成后必须自检字数，不在 10-12 字范围内则重写
```

#### 改进 2：禁止 emoji/符号编号分点

**现状问题**：round2 有 3/5 篇使用 1️⃣📌▫️• 等符号分点，工具号感严重；round3 仍有 test_05 使用 1️⃣-5️⃣ 编号。

**改进方案**：
```
【分点过渡要求】
- ❌ 禁止使用：1️⃣2️⃣3️⃣、①②③、▫️•📌、📌等符号或emoji作为编号
- ❌ 禁止使用：「---」分隔线、「>>>」箭头等装饰符号
- ✅ 必须使用：纯文字过渡词
  · 开头第一点用：「先说说/首先/第一」
  · 中间点用：「还有/另外/除此之外/第二点是」
  · 最后一点用：「最后/还有一点很重要/想特别提醒的是」
- 每个要点用独立段落展开，段落之间空行
- 如果要使用 emoji，只能作为章节情绪图标（非编号），如🌱代表兴趣、📚代表学习，且每个 emoji 后必须跟文字标题
```

#### 改进 3：强制第一人称故事/个人经历

**现状问题**：干货类话题容易变成通用建议罗列，缺少「我」的真实感；round2 有 3/5 篇个人故事感不足。

**改进方案**：
```
【叙事视角要求】
- 必须以第一人称「我」叙述，身份是学长/学姐/过来人
- 正文开头必须包含 1 个具体的个人经历或观察：
  ✅ 好例子：「去年我负责社团招新，印了500张传单发出去，结果一半进了垃圾桶」
  ✅ 好例子：「我大一的时候也是社恐，新生聚会缩在角落不敢说话」
  ❌ 坏例子：「很多同学问我社团怎么招新」（空泛，没有「我」）
  ❌ 坏例子：「作为辅导员，我带过三届学生」（如果后面不用辅导员视角展开则不要用）
- 中间每个要点至少穿插 1 处个人感受/具体观察/身边人的例子
- 结尾回到「我」的感受或建议，保持视角统一
```

#### 改进 4：正文字数兜底 + 展开深度要求

**现状问题**：3/15 篇正文不足 500 字，最短仅 401 字；清单体下每个点只有 2-3 句，展开不够。

**改进方案**：
```
【正文长度与展开要求】
- 正文总字数：严格 500-800 字
- 每个要点（建议/观点）必须展开至少 3-4 句：
  · 第1句：观点/建议是什么
  · 第2句：为什么这么说（可结合个人经历或数据）
  · 第3句：具体怎么做（操作细节）
  · 第4句（可选）：个人感受或补充提醒
- 生成后请统计字数：
  · 如果 < 500 字：在薄弱段落补充个人经历或具体细节，不要凑废话
  · 如果 > 800 字：删除重复表述，保留最有价值的内容
```

### 6.2 中优先级改进（提升质量上限）

#### 改进 5：标题与内容数量一致 + 标题多样性

**现状问题**：test_01(r3) 标题写「5个技巧」但正文只展开 3 个；部分标题模板化（「N个XX」「必看」出现频率高）。

**改进方案**：
```
【标题要求】
- 3 个备选标题，长度 12-25 字
- 如果标题中出现「N个建议/技巧/真相/心法」，N 必须与正文实际展开的要点数量一致
- 标题风格要多样，3 个标题分别尝试不同钩子类型：
  · 痛点反常识型：「别急着脱单！大学恋爱最该明白的事」
  · 身份共鸣型：「社恐新生进｜不用逼自己外向也能交到朋友」
  · 利益承诺型：「社团招新没人来？这几招让摊位挤爆」
- 避免三个标题都是「N个XX」的模板
```

#### 改进 6：绝对化词引导替换（非硬禁）

**现状问题**：「最」字出现频率较高，但多数语境自然（如「最重要的底气」「最放松的状态」），不影响阅读。

**改进方案**：
```
【用词建议】
- 以下词汇尽量替换，但如果语义需要可以保留（不做硬性禁止）：
  · 「最」→「很/相当/我个人觉得特别/…之一」
  · 「保证」→「亲测有效/我身边试过的都说有用」
  · 「必须」→「建议/真心觉得要/最好」
  · 「第一」（当序数用可以保留，如「第一点」）
```

#### 改进 7：引导主动填写 risk_notes（参考 r3 test_03）

**现状问题**：15 篇中仅 test_03(r3) 主动输出 risk_notes，其余均为空数组，但涉及金钱、学业、感情的话题应该有适当风险提示。

**改进方案**：
```
【风险提示（risk_notes）填写要求】
- 以下情况请主动填写风险提示，每条 1-2 句话：
  · 涉及金钱/创业/投资：提醒资金风险，不要拿生活费/学费试水
  · 涉及学业选择：提醒以学业为重，不要盲目跟风
  · 涉及感情建议：提醒每个人情况不同，仅供参考
  · 涉及社交技巧：提醒真诚最重要，不要刻意讨好
- 其他话题如果没有需要特别提醒的风险，可以留空数组
```

### 6.3 低优先级改进（锦上添花）

#### 改进 8：emoji 用法参考 test_04(r3) 模式

**改进方案**：如果使用 emoji，参考 test_04(r3) 的情绪图标模式——🌱代表兴趣社团、📚代表课堂、🤝代表志愿协作、🏠代表室友，emoji 后必须跟文字标题，不能单独用 emoji 编号。

#### 改进 9：加入 few-shot 示例段落

**改进方案**：将 test_02(r3)（校园恋爱）中的 1-2 个优秀段落作为 few-shot 示例写入 system prompt，锚定纯文字、有故事感、金句自然的风格标杆。

## 八、综合结论

### 8.1 v1 版本总体评价

**content-generate-xhs v1 版本完成 3 轮 × 5 选题 = 15 条输出测试，整体达标，具备基础可用性。**

| 评估维度 | 评级 | 说明 |
|---------|:---:|------|
| 代码稳定性 | ✅ 优秀 | 15 次执行零失败，JSON/字段完整性 100% |
| 格式合规性 | ✅ 良好 | 93% 字段达标，cover_text 是主要失分项 |
| 内容质量 | ✅ 良好 | 均分 19.5/25，15/15 过通过线，最高 24 分 |

### 8.2 核心结论

1. **v1 Prompt 已验证技能的基本可行性**：15 条输出全部成功，文案整体质量符合小红书「学长/过来人分享」的调性要求
2. **v1 Prompt 的主要短板是风格一致性**：同一 Prompt 在不同主题下表现波动大（16-24 分），干货类话题需要更强的风格约束
3. **cover_text 字数是最需要修复的硬伤**：47% 的篇目不合规，且持续三轮未解决，必须在 v2 中通过示例+自检解决
4. **v2 改进方向明确**：8 项改进建议按优先级排列，高优先级 4 项解决后预计均分可从 19.5 提升至 21+
5. **test_02 round3（校园恋爱 24 分）可作为 v2 的风格标杆**：纯文字、有故事、有金句、有温度，是理想输出的范本
