# 郝乐瑾：自动素材采集与平台样本库

本文所称三个平台固定指：公众号、抖音、小红书。

## 负责目标

为项目持续提供真实、有效、可追溯的平台素材，解决当前“已有采集模块，但有效选题不足、平台样本不完整”的问题。

这部分是后续平台偏好分析和流量研究的前置任务。只有素材来源稳定、样本数量足够，后续生成风格和平台规律分析才有可靠依据。

## 当前已有基础

- 已有抖音和小红书热点采集基础，公众号素材采集尚未覆盖。
- 已有热点去重、热度整理、相关性筛选和内容角度建议。
- 已有飞书每周热点素材卡片和“采纳”入口。
- 已有采集失败时的降级结果。

当前基础仍不能视为任务完成。仓库保留的运行结果中出现过“50条原始热点、0条有效选题”，抖音和小红书数据源出现过连接失败，公众号素材采集仍需补齐。

## 具体任务

- 保证公众号、抖音和小红书素材能够持续、稳定地进入项目。
- 优先解决有效选题数量不足的问题。
- 将采集范围从热点标题扩展为能够体现平台偏好的内容样本。
- 每条样本应包含标题、正文摘要、封面、标签、发布时间、来源和可获取的公开互动表现。
- 清理重复、无关、失效和明显低质量的素材。
- 为每条推荐选题保留来源、热度、建议角度和推荐平台。
- 明确区分实时采集、历史缓存和采集失败，不使用模拟内容冒充实时结果。
- 保证单个平台失败时，其他平台已经获得的素材仍然有效。
- 为平台偏好分析提供结构统一、可以直接使用的样本库。

## 交付内容

- 公众号、抖音和小红书的自动素材采集结果。
- 每周可供审核的有效选题列表。
- 能够支持语言和视觉分析的平台内容样本库。
- 每条素材的来源、时间和质量状态。
- 采集失败、数据不足和平台降级情况说明。

## 完成标准

- 每次有效运行能够提供8至10条与校园和社团运营相关的选题。
- 公众号、抖音和小红书都有足够数量的真实样本用于后续偏好分析。
- 所有素材都可以追溯到来源和采集时间。
- 无效、重复和无关内容不会进入推荐选题。
- 某个平台不可用时，不影响其他平台结果。
- 采集结果无需再次人工整理即可交给平台偏好分析。

- 向刘俊清提供完整素材和推荐选题，用于接入飞书审核和内容生成流程。

## 不在本任务范围内

- 不负责决定三个平台最终采用什么语言风格或画面风格。
- 不负责发布后的流量数据回收和推荐机制结论。
- 不负责修改整个 AgentLoop 主流程。
