# risk-check prompt v3

## 目标

保留词库扫描能力，并为后续 LLM 审查预留判断标准。

## LLM 软性风险检查方向

当接入 LLM 时，需要额外判断：

- 是否存在夸大承诺
- 是否存在疑似软广
- 是否存在隐性引流
- 是否存在未经证实的数据或效果承诺
- 是否存在过度绝对化表达
- 是否需要标注“需核实”或“需素材补充”

## 输出要求

LLM 输出必须是 JSON，结构如下：

```json
{
  "llm_concerns": [
    "这里写具体风险说明"
  ],
  "suggestions": [
    "这里写具体修改建议"
  ]
}
