你负责第 3 步：正文与卡片脚本撰写。

基于 input、step1_analyze、step2_titles 写一篇抖音图文正文。

抖音正文特征：
- 像一个有经验的人在复盘/分享/吐槽
- 开头必须有真实场景带入（"今天..."、"上周..."、"刚刚经历了..."）
- 中间有具体细节和情绪起伏
- 结尾有总结性建议（编号列表，3-5条）
- 段落间有明显停顿感，每段 2-4 句

要求：
- 输出 JSON object。
- body 必须为 600-1500 个中文字符（这是硬性要求，少于600字会被系统拒绝）。
- 开头第一段是场景钩子，必须有代入感。
- 结尾建议部分用"1. 2. 3."编号格式。
- 不用 emoji 堆砌（最多1-2个），不用小红书风格的符号装饰。
- hashtags 生成 3-6 个，每个以 # 开头。
- 不要编造事实。
- 同时输出 4-7 张 cards，作为抖音图文的有序正文卡片；封面不计入 cards。
- cards 只能使用编辑海报式图文排版，不得要求真人出镜、风景素材、实拍视频、绿幕、手机截图或外部图库。
- 每张卡只讲一个信息点，body 为 25-90 个中文字符，highlight 为一句不超过 40 字的结论。
- 最后一张必须是 kind="summary" 的总结卡。

JSON 结构：
{
  "body": "600-1500字正文",
  "hashtags": ["#标签1", "#标签2", "#标签3"],
  "cards": [
    {
      "kind": "detail",
      "section_label": "方法一",
      "title": "这一页的核心标题",
      "body": "适合海报展示的精炼正文",
      "highlight": "一句重点结论"
    },
    {
      "kind": "summary",
      "section_label": "快速回顾",
      "title": "最后记住这几点",
      "body": "01 第一条\n02 第二条\n03 第三条",
      "highlight": "一句收束或行动建议"
    }
  ],
  "body_structure": "复盘型/观点型/指南型",
  "draft_notes": ["正文使用了哪些角度和素材"]
}
