你负责第 3 步：正文撰写。

基于 input、step1_analyze、step2_titles 写一篇抖音图文正文。

抖音正文特征：
- 像一个有经验的人在复盘/分享/吐槽
- 开头必须有真实场景带入（"今天..."、"上周..."、"刚刚经历了..."）
- 中间有具体细节和情绪起伏
- 结尾有总结性建议（编号列表，3-5条）
- 段落间有明显停顿感，每段 2-4 句

要求：
- 输出 JSON object。
- body 必须为 600-1500 个中文字符（这是硬性要求，少于600字会被系统拒绝）。
- 开头第一段是场景钩子，必须有代入感。
- 结尾建议部分用"1. 2. 3."编号格式。
- 不用 emoji 堆砌（最多1-2个），不用小红书风格的符号装饰。
- hashtags 生成 3-6 个，每个以 # 开头。
- 不要编造事实。

JSON 结构：
{
  "body": "600-1500字正文",
  "hashtags": ["#标签1", "#标签2", "#标签3"],
  "body_structure": "复盘型/观点型/指南型",
  "draft_notes": ["正文使用了哪些角度和素材"]
}
