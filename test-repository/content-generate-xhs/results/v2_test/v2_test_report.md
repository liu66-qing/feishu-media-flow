## 七、v2 遗留问题与 v3 改进建议

### 7.1 核心遗留问题：段落长度不稳定

段落长度是 v2 最突出的稳定性问题，表现为：
- 情感类话题（test_02 恋爱）三轮段落数稳定在 19-24 段
- 干货类话题（test_01/03/04）段落数波动极大（6-24 段）
- 模型在信息密度高时倾向于把一个要点的完整论述塞进一个段落

根因分析：
- v2 的 few-shot 示例段落有 3-4 句，比理想的 2-3 句偏长
- Prompt 说「每段不超过3句」但未给出段落数量目标
- 自检清单没有段落数检查项
- 模型在「内容充实」和「段落简短」之间优先选择前者

### 7.2 v3 Prompt 改进建议

**高优先级**：
1. **强化段落拆分约束**：
   - 明确要求「正文必须 20-30 个段落」
   - 「每个要点必须拆成 2-3 个小段落（观点段→展开段→例子/感受段）」
   - 给出严格 2 句/段的 few-shot 示例
2. **自检清单新增段落数检查**：要求生成后统计段落数，不足 20 段时主动拆分长段落

**低优先级**：
3. cover_text 自检增加明确字数计数提示（如「生成后逐字计数」）
4. 自检清单增加标点完整性检查（避免句子粘连）

## 八、综合结论

### 8.1 v2 版本总体评价

**content-generate-xhs v2 版本完成 3 轮 × 5 选题 = 15 条输出测试，硬指标通过率 99%，肉眼均分 20.4/25，相比 v1 全面提升，已达到生产可用水平。**

| 评估维度 | v1 评级 | v2 评级 | 说明 |
|---------|:---:|:---:|------|
| 代码稳定性 | ✅ 优秀 | ✅ 优秀 | 15 次执行零失败 |
| 格式合规性 | ✅ 良好（93%） | ✅ **优秀（99%）** | 仅 1 项 cover_text 警告 |
| 内容质量 | ✅ 良好（19.5） | ✅ **良好（20.4）** | 均分提升 0.9 分 |
| 风格一致性 | ⚠️ 合格（工具号感） | ✅ **良好（零符号编号）** | 第一人称 100% 覆盖 |
| 排版稳定性 | ✅ 良好 | ⚠️ **合格（段落波动）** | 干货类段落长度不稳定 |
| 安全合规 | ⚠️ 合格 | ✅ **良好** | risk_notes 100% 填写 |
| 生产可用性 | ✅ 可用 | ✅ **良好可用** | 偶需微调段落和 cover_text |

### 8.2 核心结论

1. **v2 的 4 项高优先级改进全部验证有效**：cover_text 示例、禁止符号编号、第一人称故事、正文字数兜底均显著改善了输出质量
2. **v2 最大的胜利是风格统一**：彻底消除了 v1 中的 emoji 编号/工具号感问题，15/15 篇都是自然的学长/学姐聊天风格
3. **段落排版成为新的核心短板**：这是 v2 唯一明显退步的维度，需要在 v3 中通过更强的段落数约束来解决
4. **v2 已具备生产可用性**：99% 的硬指标通过率和 20.4 的均分意味着大部分输出可直接使用，仅需偶尔微调段落或 cover_text
5. **v2 test_02 round3（23分）是新的标杆**：在情感浓度、金句质量、人文关怀（心理咨询中心）上接近 v1 标杆（24分）水平
6. **test_05 学习就业主题进步最大**：从 v1 的工具号重灾区变成了稳定的真诚分享，验证了 Prompt 改进的精准性
