# 封面图生成效果改进对比说明

## 概述

本文档记录了封面背景图生成的改进方案，从原有的纯色方块背景升级为 AI 生成的真实场景背景图。

---

## 原有方案

### 问题

- 使用纯 HTML 模板截图，效果简陋
- 背景为纯色方块（如`#FF6B6B`红色）
- 视觉吸引力不足，不符合小红书用户的审美习惯

### 实现方式

```python
variables = {
    "bg_color": "#FF6B6B",  # 固定纯色
    "accent_color": "#FFFFFF",
}
```

---

## 新方案

### 核心思路

1. **AI 生成背景底图**：使用 DashScope wanx2.1-t2i-turbo / qwen-image-2.0-pro 生成高质量场景背景
2. **文字后期叠加**：由 Pillow/HTML 负责文字渲染，避免 AI 生图模型画文字乱码的问题
3. **智能 Prompt 生成**：根据选题内容自动匹配最合适的场景风格

### Prompt 设计策略

#### 通用结构

所有 Prompt 都遵循以下结构：

```
3:4 竖版构图，小红书封面背景，[风格关键词]，[场景关键词]，画面整体 [色调关键词]，
[光影关键词]，画面 [留白位置] 留足大面积干净的低对比度留白区域，
无任何文字、字母、数字、水印、logo，高清 8K，细节细腻，氛围统一，元素不杂乱，
适合后期叠加文字
```

#### 关键原则

1. **不要求生成文字**：明确标注"无任何文字、字母、数字、水印、logo"
2. **强调留白区域**：指定留白位置（中心/上半部分），便于后期排版
3. **适配国内大模型**：全部使用中文描述，符合 DashScope wanx/qwen 模型的输入规范
4. **低对比度柔和光影**：确保背景不会与前景文字冲突

## 降级策略

当上下文中找不到 `step1_analyze` 数据时，系统会执行降级策略：

### 优先顺序

1. **从 step1_analyze 提取 hook/value** → 调用 `_extract_style_keywords()` 获取风格关键词
2. **没有 step1_analyze，但有 topic** → 调用 `_infer_prompt_from_topic()` 基于 topic 推断
3. **都没有** → 返回默认 ins 简约风书桌背景

### 默认 Prompt

```
3:4 竖版构图，ins 简约风背景图，干净浅木色书桌场景，整体奶白 + 浅棕暖色调，
柔和窗边漫反射自然光，画面中心留足大面积干净的低对比度留白区域，
无任何文字、字母、数字、水印、logo，高清 8K，细节细腻，氛围统一，
元素不杂乱，适合后期叠加文字
```

## 代码实现位置

### WorkflowService 新增方法

- `WorkflowService._generate_cover_prompt_from_context(context_data)` - 主入口，生成最终 Prompt
- `WorkflowService._extract_style_keywords(hook, value, topic)` - 提取场景风格关键词
- `WorkflowService._build_prompt(style_keywords, topic)` - 构建完整 Prompt
- `WorkflowService._infer_prompt_from_topic(topic)` - 基于 topic 的降级推断

### 调用链路

```
WorkflowService.generate_job()
  └─> WorkflowService._compose_image_for_content()
       └─> WorkflowService._generate_cover_prompt_from_context()
            ├─> Step1: 检查 context_data 是否存在
            ├─> Step2: 查找 step1_analyze 结果
            ├─> Step3: 提取 best_angle 的 hook/value
            ├─> Step4: 调用_extract_style_keywords()
            └─> Step5: 调用_build_prompt() 组装最终 Prompt
```

## 预期效果

- **视觉质量提升**：从纯色块升级为真实场景背景，更符合小红书用户的审美习惯
- **场景匹配度提高**：不同类型的选题自动匹配对应风格的背景
- **留白更合理**：预留清晰的空间用于后期文字排版
- **兼容性保证**：不影响现有流程，仅当设置 `use_ai_image=True` 时启用新功能

---

## 注意事项

1. AI 生图可能消耗额外时间和 API 调用成本，建议在非高峰期使用
2. 如果 AI 生图失败，系统会自动回退到 HTML 纯色背景方案
3. Prompt 中严禁出现"小红书"字样，避免触发平台敏感词过滤
4. 所有 Prompt 都应包含"适合后期叠加文字"的要求，确保 AI 理解意图
