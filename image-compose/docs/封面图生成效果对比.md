# 封面图生成效果对比：改进前 vs 改进后

## 改进概述

| 维度 | 改进前 | 改进后 |
|------|--------|--------|
| 代码架构 | prompt 逻辑散落在 workflow.py | prompt 逻辑集中在 image-compose/main.py |
| 默认模式 | HTML 模板截图（`use_ai_image=False`） | 可选 AI 生图（`use_ai_image=True` 时启用） |
| Prompt 质量 | 简单拼接，含"小红书"敏感词 | 基于 cover_prompt_guide.md，8 场景自动匹配 |
| 风格分类 | 无分类，统一模板 | 8 个场景（学习/校园/美食/生活/穿搭/活动/通知/情绪） |
| 背景效果 | 纯色方块 / 简单渐变 | 真实场景感背景图（书桌、咖啡馆、校园等） |

---

## 架构改进

### 改进前：prompt 逻辑侵入 workflow.py
```
workflow.py
  ├─ _generate_cover_prompt_from_context()  # 100+ 行
  ├─ _extract_style_keywords()              # 150+ 行
  ├─ _build_prompt()                        # 20 行
  └─ _infer_prompt_from_topic()             # 100+ 行

image-compose/main.py
  └─ build_ai_prompt()  # 简单拼接
```

**问题**：
- workflow 层不应该感知 prompt 细节
- 4 个辅助方法增加 workflow 复杂度
- 难以独立测试和迭代 prompt 策略

### 改进后：零侵入 + 职责分离
```
workflow.py
  └─ _compose_image_for_content()  # 原始代码，零修改
      └─ 传递 title/subtitle → image-compose

image-compose/main.py
  ├─ SCENE_STYLE_MAP        # 8 场景风格映射
  ├─ SCENE_KEYWORDS         # 场景关键词库
  ├─ infer_scene_from_text() # 场景推断
  └─ build_ai_prompt()      # 专业 prompt 组装
```

**优势**：
- workflow.py 零侵入，保持原有职责
- image-compose 作为独立技能包，可单独测试和迭代
- prompt 策略集中管理，易于扩展新场景

## 改进前 Prompt 示例

### 选题：「考研复习时间规划」

```
小红书封面背景图，考研复习时间规划，氛围感摄影风格，无文字，无水印，高质量，竖构图
```

**问题**：
- 包含"小红书"字样，可能触发平台敏感词
- 所有场景都用同一个模板，缺乏差异化
- 没有场景风格匹配
- 没有强调留白、低对比度等关键约束

## 技术改进细节

### 1. 零侵入架构
**改进前**：workflow.py 增加 370+ 行 prompt 逻辑
**改进后**：workflow.py 零修改，所有改进在 image-compose/main.py

### 2. 移除"小红书"敏感词
**改进前**：`小红书封面背景图`
**改进后**：`3:4竖版封面背景底图`

### 3. 场景自动匹配
**改进前**：所有场景用同一个模板
**改进后**：8 个场景风格映射，根据 title/subtitle 自动匹配

### 4. 强化 prompt 约束
**改进后 prompt 包含**：
- ✅ `无任何文字、字母、数字、水印、logo`
- ✅ `绝对不要生成任何文字`
- ✅ `画面中心留大面积干净低对比度留白区域`
- ✅ `装饰元素只分布在画面四周`
- ✅ `像博主随手拍的照片`
- ✅ `不要清晰人物正脸`
- ✅ `高清8K细节真实`
